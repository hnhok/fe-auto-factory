# FE-Auto-Factory v3.4.0 发布公告 🚀

> **从「提效工具」到「企业资产治理方案」的华丽转身。**

在这次意义重大的 v3.4.0 版本更新中，FE-Auto-Factory 不再仅仅满足于“代码写得快”。我们不仅深耕了 AI 视觉识别的精度与成本控制，更首次引入了**企业级代码覆盖率统计**与**工业级视图治理模板**，使得生成的每一行代码都具备直接上生产环境的质量。

---

## 🌟 核心更新亮点

### 1. 🏗️ 工业级视图模板升级 (Skeleton & Error State)
生成的代码不再是“半成品”。官方内置模板现已全面支持：
- **骨架屏自愈**：数据加载期间自动展示骨架屏，告别白屏。
- **全局错误容灾**：内置异常捕获 UI 与重试机制。
- **空态管理**：数据为空时自动展示 Vant 语义化反馈。
- **无感追踪**：自动注入 `data-track-id`，适配企业级埋点方案。

### 2. 🔄 批量逆向工程 (Batch Swagger Sync)
将后端同步提速 10 倍！
- 运行 `sync --swagger` 时，工厂会自动识别 Swagger 文档中的所有业务实体。
- **一键生成矩阵**：自动为每个实体创建独立的 Page Schema 草图，不再需要人工逐个指定模型名。
- **智能避让**：已存在的图纸会受到保护，确保您的自定义配置不被覆盖。

### 3. 📸 视觉快照沉淀系统 (Vision Snapshot)
让 AI 识别接近「零成本」：
- **三重过滤机制**：MD5 精确命中（0 Token）→ 语义相似度匹配（Jaccard ≥ 45%）→ AI 视觉补位。
- **资产复用**：识别过的设计稿自动存入 `.factory/snapshots/`，供全团队共享。
- **快照管理**：新增 `snapshot list` 与 `snapshot delete` 命令，让视觉资产可查、可控。

### 4. 📊 资产清单与覆盖率统计 (The `list` Command)
管理层视角的工程化视图：
- **覆盖率量化**：通过 `npx fe-factory list` 直观查看当前项目中有多少比例的功能是由 Schema 驱动生成的。
- **清单审计**：自动生成 `.factory/generated-files.json`，追踪每一个 Generated Page 的生命周期。
- **孤儿 Schema 预警**：一键找出定义了图纸但尚未落地的功能点。

### 5. 🛡️ 安全、透明与合规
- **Dry-Run 模式**：运行 `generate --dry-run` 即可在写盘前预览完整的文件变动清单。
- **组件复用检测**：生成时自动扫描 `src/components/`，已有组件优先复用，杜绝重复造轮子。
- **结构化退出码**：为 CI/CD 优化，提供 Schema 错误 (2)、驱动缺失 (3)、逻辑崩溃 (4) 等精准状态。

---

## 🔧 开发者体验改进
- **DEBUG 模式**：通过 `DEBUG=fe-factory:*` 开启详细的内核执行堆栈。
- **开放 Features Schema**：支持任意自定义布尔 Feature Flag。
- **版本号自动同步**：所有 Banner 和日志版本号均动态步进，与 `package.json` 保持严格一致。
