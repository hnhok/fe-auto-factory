# 项目总结与未来规划 (Roadmap)

`fe-auto-factory` 从一个简单的脚本演进至今，已成为一套具备 **Schema 驱动、多端适配、AI 赋能** 的工业级前端代码生产引擎。

## 📝 项目阶段性总结

通过 **v2.0 - v2.3** 的高速迭代，我们完成了以下核心资产的构建：

1.  **架构解耦 (Decoupling)**：实现了核心引擎与业务模板的物理分离，支持云端动态拉取模板。
2.  **多端适配 (Adapter Pattern)**：内建了对 Vue3 (Vant/Element Plus) 和 React (Ant Design) 的原生支持，实现了架构中立。
3.  **安全性飞跃 (AST Engineering)**：引入了 `ts-morph` 抽象语法树解析技术，让路由注入等源码级操作从“字符串替换”进化为“精密手术”。
4.  **AI 生产力落地 (VLM Vision)**：打通了从「UI 设计稿截图」到「YAML Schema」再到「工程代码」的视觉闭环。

---

## 🚀 中期规划 (Mid-term: 2026 Q2 - Q3)

**目标：从“指令驱动”向“可视/可插拔”演进。**

*   **[Done] 驱动沙箱系统 (Driver Sandbox)**:
    *   实现了本地驱动优先加载机制。开发者可以在项目 `.factory/drivers/` 下定义私有驱动，实现对中央工厂引擎的无损扩展。
*   **[Done] 增量原子手术 (Incremental Surgery)**:
    *   API 生成已切至 AST 增量合并模式。即便文件已存在，工厂也能通过分析语法树，安全地追加新导出的接口，不再暴力覆盖。
*   **[In Progress] 可视化图纸搭建工作台 (Low-Code GUI)**:
    *   启动 `fe-factory ui` 生成本地 Web 界面，支持通过拖拽组件、点选接口来生成 YAML Schema。
*   **[稳健] 全量 AST 突变覆盖**：
    *   将 Hook 注入、API 导出、Store 注册等所有文件修改逻辑全部切换为 AST 操作，确保 100% 的代码合并成功率。

---

## 🪐 远期规划 (Long-term: 2026 Q4+)

**目标：迈向“自治理”与“深度 AI 融合”的终极工厂。**

*   **[智能] 局部 RAG 与组件复用感知**：
    *   AI 引擎将自动扫描项目现有的 `src/components` 资产库。在生成代码时，AI 会优先尝试复用已有的业务组件，而不是生成重复的 HTML 标签。
*   **[闭环] 自我纠错与闭环编译 (Self-Healing)**：
    *   工厂不仅生成代码，还将自动运行 `lint` 和 `type-check`。如果发现生成的代码有错，将报错信息反馈给大模型进行二轮修正，直到代码通过编译。
*   **[生态] 设计稿零摩擦同步 (Design-to-Code Sync)**：
    *   开发专用 Figma 插件，实现从设计图直接推送原子级变量与布局到前端仓库，达成“设计即代码”的极致流畅感。

---

> **使命声明**：
> 我们的目标不是取代前端工程师，而是将他们从繁琐的流水线搬砖中解放出来，专注于更具创造性的架构设计与核心业务逻辑。
