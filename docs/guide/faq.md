# 故障排除指南与常见问题 (FAQ & Troubleshooting)

虽然 `FE-Auto-Factory` 在架构上经历了无数次打磨，并在底层引入了工业级的 `AJV` 断言与 `ts-morph` AST 操作，但不同的机器环境和使用姿势仍可能导致一些小问题。

这里收录了日常开发和共建中大家最容易碰到的“水坑”。

---

## 1. Schema 编译解析阶段

### ❓ 报错 `Ajv validation failed` / 图纸不合法

**表现**：运行 `generate` 后控制台直接红字报错，说 `features` 之类的格式不对。
**原因**：你在 YAML 文件里写错层级了，或者类型不规范。
**解决办法**：
- 严格对照 `[ProjectRoot]/schemas/page.schema.json` 这个用来制约语法的金科玉律进行修复。
- 比如：你是不是把 `models` 下面的类型定义漏掉了，或者字段名称敲错成了 `modelsx`。

### ❓ NotFound 找不到 `preset` 或渲染沙箱 (Driver)

**表现**：警告说 `Cannot find driver for preset: null`
**原因**：你可能是在老项目中手动创建了 `.factory/config.json`，但忘记里面加上 `"preset": "vue3-element-admin"` 这句。引擎在路由转发时找不到应该用哪套模板渲染。
**解决办法**：在项目的根目录下把你的脚手架血统写明白。

---

## 2. 核心 AST 无损缝合阶段 

### ❓ AST 注入路由失败（警告：无法动态修改 `router/index.ts`）

**表现**：生成的组件都有，但最后一步红字报：“未能在 `router/index.ts` 中找到匹配的 AST 节点插入位”。
**原因**：
1. 你的路由文件写得太“非主流”了！我们底层的检测算法会去找 `const routes = [...]`，或者是 `createRouter({ routes: [...] })`。如果你的老项目把数组名故意藏进极其深层的闭包或者改名成了 `appViewPathList`。
2. 你项目根目录的 `tsconfig.json` 对于 `@/` （或者你用其他的 alias）配置失效了，导致文件寻址死锁。
**解决办法**：
- 在代码里老老实实用语义化的导出名。
- 如果真的太旧不能改：手工把这段路由挂在老文件里吧，毕竟 `generate` 并不会由于这一项失败就删除上面帮你建好的组件。

### ❓ 为什么我在 Hooks 里手写的逻辑被删了？！不是吹牛批无损留存吗？

**表现**：二次更新增加业务接口调用时，发现之前在 `hooks/useXXX.ts` 中写的方法不见了。
**原因**：你在写业务方法的时候，**没有写在约定的被保护的插槽里！**
**解决办法**：
工厂为了确保识别，我们会在生成的文件里预埋注释 `// [FACTORY-HOOK-CUSTOM-START]` 与结尾 `END`。只要你在这个花括号里添加的方法，或者在 `state: reactive({})` 原有的挂载点后面跟上的新数据，它是**绝对**会被引擎提早在内存中打好快照留下的。写在外部就会被当做冗余被剔除或覆盖了。

## 3. GitHub 挂载与网络环境

### ❓ 初始化 `factory init` 时，克隆拉取失败死锁？

**表现**：转圈圈十几分钟没动静，最后爆出 `git fetch` 或 `ssh` 问题。
**原因**：由于我们的模板引擎高度去中心化，它强依赖你们电脑到 `github.com`（如果是官方开源架构预设库的话）的网络链路。这可能是 Git 没有配置代理，或者是 DNS 污染导致。
**解决办法**：
- 国内开发者检查代理：`git config --global http.proxy http://127.0.0.1:xxxx` （你的端口）。
- 如果你在公司内网物理隔绝：联系架构组管理员！将官方模版的源码打包进你们自己的 Gitlab 库！然后在 `scripts/factory.js` 内部把 `TEMPLATE_REPOS` 的链接改向你们的内网地址，这是 `init` 天然支持的做法。

## 4. 彻底重置和暴力覆盖

### ❓ 怎么强行让它干掉旧手写代码和 AST，完全给我一份新拷贝？

**解决办法**：
简单粗暴地把你之前生成的那个业务对应的 `src/views/XXX` 目录、`src/api` 和 `src/store` 下对应的源文件物理删除，甚至去路由树上把它的节点干掉。然后重新执行 `generate --schema`。此时引擎因为检索不到原有的文件作为 AST 解析底座，将立即进入**全新降维出装模式**！
