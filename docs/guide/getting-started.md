# ⚡ 入门指南与架构接入

欢迎使用企业级前端自动化生成工厂！为了让你以最快、最平缓的曲线感受到由 **Schema / 图像驱动开发** 的震撼开发体验，请根据你当前所属的环境，选择对应的接入路径。

---

## 路径 A：拔地而起（创建全新业务项目）

如果你接到了一项新需求，需要从零搭建一套符合公司规范、代码极其标准优雅的 Vue3 移动端 / B端 站点。这套工厂自带**保姆级脚手架**。

### 第一步：召唤你的专属工厂
进入你的工作区目录，直接执行以下命令，开启完全互动式的创建流：

```bash
npx fe-factory
```
或直接带上你要的新项目名：
```bash
npx fe-factory init my-awesome-app
```

> **🌍 全新一代的云端多端模板矩阵引擎**
>
> 我们不再硬编码任何面条模版。工厂引擎目前直接挂载了三大云端独立架构仓（Remote Git Templates）。你会看到一个极其优雅的选型菜单：
> - 📱 **移动端 H5 业务模板**：Vue3 + Vant 4 的极致轻度体验底座。
> - 💻 **PC 中后台管理系统 (Vue)**：Vue3 + Element Plus 的旗舰级管理台大厦骨架。
> - ⚛️ **PC 中后台管理系统 (React)**：纯血 React + Ant Design 驱动的中控台前沿利器。
>
> 一旦选中，脚手架会瞬间从云端执行**浅克隆 (Shallow Clone)**，并在落实在你本地后进行 **Git 无痕剥离**，移交给你一个最完美纯净的一手本地分支！
> 不仅如此，所选模板的「血统指纹(preset)」会自动烙印在新生项目的 `.factory/config.json` 里。接下来的所有 AI 智能化编码动作，都会根据这个血统为你做「千人千面」的代码生成转换。

### 第二步：进入项目与运行

只需按常规前端项目启动即可，你的开发环境已就绪！

```bash
cd my-awesome-app
npm install      # 拉取模板所有的基础包
npm run dev      # 启动本地开发服务器
```

*(此时工厂的第一阶段任务完成，你可以前往“开发实战篇”学习如何光速造页面了！)*

## 路径 B：旧貌换新颜（在已有老项目中植入工厂引擎）

如果你手头已经有了一个公司里的老项目（目前建议是 Vue 3 栈或者相近架构），你想在后续开发新页面时，不再苦逼地手工复制粘贴，而是享受到 AI 写代码的红利，你只需两步把工厂引擎“植入”进去。

### 第一步：作为开发依赖安装到你的工程

在你的老项目根目录下执行安装：

```bash
npm install @hnhok/fe-auto-factory@latest -D
```

### 第二步：创建脚手架目录体系

为了让工厂引擎认识你的老家，你需要按规范创建好它的工作台：

1. 在项目根目录，手动建一个 `.factory` 隐藏文件夹，并在里面放入 `config.json` 文件：
   ```json
   {
      "projectName": "your-legacy-app",
      "factoryVersion": "2.x.x",
      "schema": {
         "pagesDir": "schemas/pages"
      }
   }
   ```
2. 在项目根目录，创建 `schemas` 和 `schemas/pages` 文件夹，这里会用来存放你后续编写的构架图纸。

如果你不确定怎么写业务，可以直接从你的终端运行 `npx fe-factory` 来呼出我们的互动式主控菜单进行验证！

## 🔄 基建迭代与红利追溯 (Update)

无论是全新项目还是老项目，工厂主库（`@hnhok/fe-auto-factory` 后续发版）存在超燃新功能时，你只需执行：

1. `npm install @hnhok/fe-auto-factory@latest -D` 更新包裹
2. 运行 `npx fe-factory update` 同步资产。

**脚手架工具非常注重演进透明度。** 每执行一次上述同步，工厂包就会自动在你当前所在的业务工程里的 `docs/CHANGELOG.md` 中追写对应的自动生成更新日志，防止团队人员信息断层。
